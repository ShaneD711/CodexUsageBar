# 结构化 App-Server 错误与语言区域边界

## 范围

CodexUsageBar 将停止通过搜索服务端英文错误文案来判断登录失败。本次修改同时为错误诊断增加请求阶段、收紧简体中文识别范围，并让所有绝对日期和时间跟随用户的系统区域设置。

本次不增加繁体中文翻译、不暴露原始服务端错误文案，也不改变仅菜单栏应用的产品范围。

## 结构化账户检查

完成 `initialize` 后，客户端先发送 `account/read`，再发送 `account/rateLimits/read`。

账户响应按结构化字段解码：

- `account == null` 且 `requiresOpenaiAuth == true` 表示用户尚未登录。
- 其他有效账户响应允许继续读取额度。
- 账户响应结构无效时，报告 `account` 阶段的响应不兼容。

账户检查与额度读取共用现有的单调时钟总截止时间，不会额外产生一轮完整超时。

## 错误模型

`AppServerPhase` 表示错误发生位置：

- `launch`
- `initialize`
- `account`
- `rate-limits`

`CodexAppServerError` 只携带安全的结构化上下文：

- 未找到可执行文件
- 启动失败
- 尚未登录
- 携带阶段的超时
- 携带阶段的连接关闭
- 携带阶段的无效响应
- 携带可选 JSON-RPC 错误码与阶段的服务端错误

JSON-RPC 响应会解析 `code`、`message` 和可选 `data`，但只保留 `code`。原始 `message` 和 `data` 不进入应用状态、复制的诊断信息或测试中的诊断输出。

`UsageFailure` 保存稳定的用户错误类别，以及可选阶段和服务端错误码。现有本地化错误提示继续根据稳定类别选择文案。

## 诊断信息

复制的诊断信息继续排除账户详情和额度数值。存在错误时增加：

```text
Category: not-logged-in
Phase: account
Error code: none
```

没有值的字段统一显示 `none`。不包含原始服务端错误文案和 `error.data`。

## 语言与区域格式

只有明确表示简体中文的标准化语言、文字或地区组合才选择简体中文：

- `zh-Hans`
- `zh-CN`
- `zh-SG`

`zh-Hant`、`zh-TW`、`zh-HK` 等繁体中文标识暂时回退英文，直到应用提供完整繁体中文翻译。

UI 文案跟随 `AppLanguage`。绝对日期和时间不再使用应用语言对应的 Locale；菜单栏和弹窗都使用 `Locale.autoupdatingCurrent`，保留用户的地区和 12/24 小时设置。

## 测试

自动化测试覆盖：

- 结构化的未登录账户响应；
- 账户验证成功后继续解析额度；
- 保留 JSON-RPC 错误码但不保留错误文案；
- 各相关阶段的超时、EOF、无效响应和服务端错误；
- 诊断信息包含类别、阶段和错误码，但不包含服务端文案；
- 简体中文标识与繁体中文回退；
- 菜单栏和弹窗使用同一个系统区域默认格式；
- 现有进程取消、总截止时间、缓存和刷新行为。

## 兼容边界

实现只依赖本机 Codex app-server 生成的 JSON Schema 中已有字段。如果未来 Codex 删除或修改 `account/read`，应用会报告带阶段的服务端错误或不兼容响应，而不会重新根据人类可读文案猜测原因。
