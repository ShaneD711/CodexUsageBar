# CodexUsageBar v0.1.1 可靠性设计

[English](2026-07-12-v0.1.1-reliability-design.md) | [简体中文](2026-07-12-v0.1.1-reliability-design.zh-CN.md)

日期：2026-07-12

## 概述

`v0.1.1` 是一次聚焦可靠性和可诊断性的更新。应用继续保持只有菜单栏和点击后的弹出面板，不增加主窗口、设置页或用量分析功能。

本次更新要让两类信息更清楚：用户当前看到的数据是否可信，以及当前运行的是哪个 CodexUsageBar 应用副本。

## 目标

- 界面根据 macOS 首选语言自动显示简体中文或英文。
- 将常见读取失败区分为可执行的错误提示。
- 在弹出面板中显示应用版本。
- 允许用户在 Finder 中定位当前正在运行的 `.app`。
- 允许用户复制经过脱敏的诊断信息，便于提交 GitHub Issue。
- 为 Codex 可执行文件查找顺序和新增支持逻辑补充自动化测试。
- 保留现有刷新、缓存和过期警告行为。

## 不在范围内

`v0.1.1` 不包含：

- DMG 安装器。
- Developer ID 签名或 Apple 公证。
- 自动检查更新或自动更新。
- 开机启动。
- 手动语言切换。
- 菜单栏显示模式设置。
- 主窗口、统计图表、Token 热力图或历史分析。
- Claude Code 支持、账号切换或通知。

这些能力不得因为实现便利而混入本版本。

## 用户体验

菜单栏继续只显示 5 小时窗口的剩余百分比和重置时间：

```text
75% 16:57
```

点击后的 320 点宽弹出面板继续显示 5 小时和 1 周两个窗口。底部状态改为：

```text
刚刚刷新 · v0.1.1                  ↻  …  ⏻
```

省略号菜单包含两个命令：

- `在 Finder 中显示`：定位当前正在运行的应用包，而不是按名称搜索其他副本。
- `复制诊断信息`：将脱敏后的纯文本诊断信息写入剪贴板。

刷新和退出继续使用图标按钮及悬停提示。新增命令放在菜单内，避免弹出面板变成设置页。

弹出面板顶部使用单行纯文字标题 `Codex剩余用量`，不显示应用图标或第二行副标题，使剩余百分比保持为主要视觉信息。

弹出面板内容与外框之间统一使用 12 点内边距。各分区独立控制垂直间距：标题到分隔线 10 点、用量区上下各 12 点、底部分隔线到操作行 8 点、操作行到底边 12 点，以匹配 macOS 原生菜单弹窗的紧凑节奏。

视觉样式使用 macOS 系统语义值：标题为 `headline`，主要用量为 `body`，底部状态为 `caption`；用量名称和百分比使用系统 `semibold`，重置时间保持常规字重。进度条、刷新按钮、更多菜单和退出按钮使用系统 `small` 控件尺寸，不指定固定字体名称或固定字号。

## 本地化

应用只支持 `简体中文` 和 `English`，启动时根据 `Locale.preferredLanguages` 的第一项选择语言。以 `zh` 开头时使用简体中文，其他情况使用英文。应用不提供单独的语言选项。

由于当前应用同时通过 SwiftPM 和自定义脚本组装，`v0.1.1` 使用小型、类型安全的 `AppLocalization` 字符串目录，不引入额外资源包。语言选择和所有用户可见字符串集中管理，不在 SwiftUI 视图中散落语言判断。

日期、时间和相对时间继续使用系统 locale 格式化，界面标签和错误说明由字符串目录提供。

## 错误模型

`UsageStore` 不再只保存已经格式化的错误文本，而是保留结构化失败类别。界面在显示时再根据当前语言生成说明。

需要区分以下状态：

| 类别 | 中文提示 | 建议动作 |
| --- | --- | --- |
| 未找到 Codex | 未找到 Codex。请先安装 ChatGPT 或 Codex。 | 安装应用或 Codex CLI |
| 未登录 | Codex 尚未登录。请先打开 Codex 并登录。 | 打开 Codex 登录 |
| 超时 | 读取 Codex 用量超时，请稍后重试。 | 手动刷新 |
| 响应不兼容 | 当前 Codex 版本返回了不支持的用量数据。 | 更新 CodexUsageBar 或提交 Issue |
| 启动失败 | 无法启动 Codex 用量服务。 | 复制诊断信息 |
| 服务错误 | Codex 用量服务返回错误。 | 手动刷新或复制诊断信息 |

如果已有成功快照，任何刷新失败都继续显示旧数据，并保留现有的过期警告。错误类别只解释本次刷新失败，不能清空可用快照。

## Codex 可执行文件解析

解析器返回 `ResolvedCodexExecutable`，其中包含可执行文件 URL 和来源类型。候选顺序保持不变：

1. `CODEX_EXECUTABLE` 环境变量。
2. `/Applications` 中的 ChatGPT 或 Codex 应用。
3. 用户 `~/Applications` 中的 ChatGPT 或 Codex 应用。
4. `~/.local/bin/codex`。
5. Homebrew 或系统常见路径。
6. `PATH` 中的 `codex`。

文件系统、环境变量、用户目录和 `PATH` 查找应可注入，使测试不依赖开发者电脑上的真实安装情况。

## 应用位置与版本

`在 Finder 中显示` 使用 `Bundle.main.bundleURL` 定位当前运行包，并通过 `NSWorkspace.activateFileViewerSelecting` 显示它。若开发环境不是标准应用包，则向上查找到最近的 `.app`；仍无法定位时，该菜单项禁用。

仓库新增根目录 `VERSION` 文件，内容为 `0.1.1`。开发构建和发布打包脚本都从该文件写入 `CFBundleShortVersionString` 与构建版本，应用界面从 `Bundle` 读取版本。脚本不得再各自写死版本号。

## 诊断信息与隐私

复制出的诊断信息使用稳定的纯文本格式，例如：

```text
CodexUsageBar: 0.1.1
macOS: 15.5
Architecture: arm64
Codex source: ChatGPT.app
Codex executable: /Applications/ChatGPT.app/.../codex
Snapshot: available, fresh
Last refresh: 2026-07-12T10:30:00+08:00
Last error: timeout
```

诊断信息可以包含：

- CodexUsageBar 版本。
- macOS 版本和 CPU 架构。
- Codex 可执行文件来源与脱敏路径。
- 是否存在快照、快照是否过期、最后刷新时间。
- 最后一次结构化错误类别。

诊断信息不得包含：

- 用户名或完整用户目录；用户目录统一替换为 `~`。
- 认证 token、`auth.json` 内容或账号标识。
- 用量百分比、重置时间或其他账户额度数据。
- 会话、提示词、工具调用或项目路径。

复制成功后不增加弹窗或通知；菜单项短暂显示 `已复制`，随后恢复，避免打断用户。

## 代码边界

- `AppLocalization`：语言选择和用户可见字符串。
- `UsageFailure`：稳定的失败类别及底层错误映射。
- `ResolvedCodexExecutable`：可执行文件 URL 与来源。
- `AppSupport`：版本、运行应用位置、路径脱敏和诊断文本生成。
- `UsageStore`：保存快照、刷新状态和结构化失败，不直接操作 AppKit。
- `UsagePopoverView`：展示状态并触发刷新、Finder 和剪贴板命令。

AppKit 调用集中在 `AppSupport`，避免把 Finder 和剪贴板细节散落到 SwiftUI 视图。

## 测试

新增或扩展测试覆盖：

- `zh-Hans`、`zh-CN` 选择中文，英文和其他语言回退英文。
- Codex 解析器保持既定优先级，并覆盖 ChatGPT.app、Codex.app、CLI 和未找到状态。
- 结构化 app-server 错误正确映射为 `UsageFailure`。
- `.app` 位置解析能区分下载版、工程构建版和非应用包运行环境。
- 诊断路径将用户目录替换为 `~`。
- 诊断文本不包含用户名、认证字段和用量值。
- `VERSION` 与打包后 `Info.plist` 版本一致。
- 现有解析、刷新与过期判断测试继续通过。

## 验收标准

`v0.1.1` 完成时应满足：

- 中文系统显示中文，其他系统显示英文，菜单栏数值格式不受影响。
- 用户能直接定位当前正在运行的应用副本。
- 用户能复制足够排查问题且不泄露个人信息的诊断文本。
- 无缓存时的主要错误给出明确原因；有缓存时仍保留最后成功数据。
- 开发包、发布包和界面显示的版本来自同一个 `VERSION` 文件。
- 菜单栏仍是应用的唯一常驻界面，没有新增主窗口或设置页。
