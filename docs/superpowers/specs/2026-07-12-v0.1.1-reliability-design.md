# CodexUsageBar v0.1.1 Reliability Design

[English](2026-07-12-v0.1.1-reliability-design.md) | [简体中文](2026-07-12-v0.1.1-reliability-design.zh-CN.md)

Date: 2026-07-12

## Overview

`v0.1.1` is a focused reliability and diagnostics update. CodexUsageBar remains a menu-bar-only app with one popover. It does not gain a main window, settings screen, or usage analytics.

The release should make two things clearer: whether the displayed usage is trustworthy and which CodexUsageBar application copy is currently running.

## Goals

- Follow the macOS preferred language with Simplified Chinese and English UI.
- Turn common read failures into distinct, actionable messages.
- Show the application version in the popover.
- Reveal the currently running `.app` in Finder.
- Copy privacy-safe diagnostics suitable for a GitHub issue.
- Add deterministic tests for Codex executable resolution and support logic.
- Preserve the existing refresh, cache, and stale-warning behavior.

## Non-Goals

`v0.1.1` does not include:

- A DMG installer.
- Developer ID signing or Apple notarization.
- Update checks or automatic updates.
- Launch at login.
- A manual language selector.
- Menu bar display preferences.
- A main window, charts, token heatmaps, or historical analytics.
- Claude Code support, account switching, or notifications.

These features must not enter this release as implementation side effects.

## User Experience

The menu bar continues to show only the five-hour remaining percentage and reset time:

```text
75% 16:57
```

The 320-point popover continues to show the five-hour and weekly windows. Its footer becomes:

```text
Updated just now · v0.1.1              ↻  …  ⏻
```

The ellipsis menu contains two commands:

- `Show in Finder`: reveal the application bundle that is currently running, rather than searching for another copy by name.
- `Copy Diagnostics`: put a privacy-safe plain-text report on the clipboard.

Refresh and Quit remain icon buttons with hover help. Secondary commands stay in a menu so the popover does not become a settings surface.

The popover uses the single-line text header `Codex Usage Remaining`, without an app icon or a second subtitle row, so the remaining percentages stay visually primary.

The popover uses 12 points of outer content padding. Section spacing is explicit: 10 points below the header, 12 points above and below usage, and 8 points between the lower divider and action row, followed by the 12-point outer inset. This matches the compact rhythm of native macOS menu popovers.

Visual styling uses macOS semantic values: `headline` for the title, `body` for primary usage, and `caption` for footer status. Usage names and percentages use the system semibold weight, while reset times remain regular. Progress, refresh, menu, and quit controls use the system small control size, with no fixed font family or point size.

## Localization

The app supports `Simplified Chinese` and `English`. It selects a language at launch from the first entry in `Locale.preferredLanguages`: identifiers beginning with `zh` use Simplified Chinese; all others use English. There is no in-app language override.

Because the app is assembled through both SwiftPM and custom scripts, `v0.1.1` uses a small type-safe `AppLocalization` string catalog instead of adding a resource bundle. Language selection and user-visible strings are centralized; SwiftUI views do not contain scattered locale checks.

Dates, times, and relative times continue to use the system locale. Labels and error explanations come from the app string catalog.

## Error Model

`UsageStore` stores a structured failure category instead of only a formatted error string. The view localizes that category when it renders.

The following states must be distinct:

| Category | English message | Suggested action |
| --- | --- | --- |
| Codex not found | Codex was not found. Install ChatGPT or Codex first. | Install the app or Codex CLI |
| Not signed in | Codex is not signed in. Open Codex and sign in first. | Sign in through Codex |
| Timeout | Reading Codex usage timed out. Try again shortly. | Refresh manually |
| Unsupported response | This Codex version returned unsupported usage data. | Update CodexUsageBar or file an issue |
| Launch failure | The Codex usage service could not be started. | Copy diagnostics |
| Service failure | The Codex usage service returned an error. | Refresh or copy diagnostics |

If a successful snapshot exists, every refresh failure preserves it and keeps the current stale-warning behavior. The failure category explains the latest refresh only; it must not clear usable data.

## Codex Executable Resolution

The resolver returns a `ResolvedCodexExecutable` containing the executable URL and a source category. Candidate precedence remains unchanged:

1. The `CODEX_EXECUTABLE` environment variable.
2. ChatGPT or Codex under `/Applications`.
3. ChatGPT or Codex under `~/Applications`.
4. `~/.local/bin/codex`.
5. Common Homebrew or system locations.
6. `codex` found through `PATH`.

Filesystem access, environment variables, the home directory, and `PATH` lookup are injectable so tests do not depend on software installed on the developer's Mac.

## Application Location and Version

`Show in Finder` starts from `Bundle.main.bundleURL` and uses `NSWorkspace.activateFileViewerSelecting` to reveal the running bundle. In a development environment that is not represented by a standard bundle, it walks upward to the nearest `.app`; the command is disabled if no application bundle can be identified.

The repository gains a root `VERSION` file containing `0.1.1`. Development and release scripts both use it for `CFBundleShortVersionString` and the build version. The UI reads its version from `Bundle`. Scripts must not maintain independent hard-coded versions.

## Diagnostics and Privacy

The copied diagnostic report uses stable plain text, for example:

```text
CodexUsageBar: 0.1.1
macOS: 15.5
Architecture: arm64
Codex source: ChatGPT.app
Codex executable: /Applications/ChatGPT.app/.../codex
Snapshot: available, fresh
Last refresh: 2026-07-12T10:30:00+08:00
Last error: timeout
```

Diagnostics may contain:

- CodexUsageBar version.
- macOS version and CPU architecture.
- Codex executable source and a redacted path.
- Snapshot availability, freshness, and last refresh time.
- The most recent structured failure category.

Diagnostics must not contain:

- A username or complete home-directory path; the home directory is replaced with `~`.
- Authentication tokens, `auth.json` contents, or account identifiers.
- Usage percentages, reset times, or other account quota values.
- Sessions, prompts, tool calls, or project paths.

Copying does not open an alert or notification. The menu item briefly changes to `Copied`, then returns to its normal label.

## Code Boundaries

- `AppLocalization`: language selection and user-visible strings.
- `UsageFailure`: stable failure categories and lower-level error mapping.
- `ResolvedCodexExecutable`: executable URL and source.
- `AppSupport`: app version, running bundle location, path redaction, and diagnostic report generation.
- `UsageStore`: snapshot, refresh state, and structured failure state; no direct AppKit operations.
- `UsagePopoverView`: renders state and invokes refresh, Finder, and clipboard commands.

AppKit calls remain inside `AppSupport` so Finder and clipboard details do not spread through SwiftUI views.

## Tests

New or expanded tests cover:

- `zh-Hans` and `zh-CN` select Chinese; English and other languages fall back to English.
- Resolver precedence and ChatGPT.app, Codex.app, CLI, and not-found outcomes.
- Structured app-server errors map to the correct `UsageFailure`.
- Application-bundle detection distinguishes downloaded, development-build, and unbundled execution paths.
- Diagnostic paths replace the home directory with `~`.
- Diagnostic reports exclude usernames, authentication fields, and usage values.
- `VERSION` matches the packaged `Info.plist` version.
- Existing parsing, refresh, and freshness tests continue to pass.

## Acceptance Criteria

`v0.1.1` is complete when:

- Chinese systems show Chinese and other systems show English without changing menu bar value formatting.
- Users can reveal the exact application copy that is currently running.
- Users can copy diagnostics that are useful for support without exposing personal data.
- Primary failures have a clear reason when no cache exists; cached data remains visible after later failures.
- Development bundles, release bundles, and UI version text derive from one `VERSION` file.
- The menu bar remains the only persistent interface, with no new main window or settings screen.
